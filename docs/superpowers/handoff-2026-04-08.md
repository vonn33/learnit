## What Was Built This Session

### Post-Wave-B cleanup (commits before `acab3f2`)

Four follow-up items were closed out before Wave C work began:

1. **DropHandler render-phase side-effect** — `onScreenToFlow` was being called during render (ref mutation). Fixed by changing the API: `DropHandler` now accepts `converterRef: RefObject<...>` and assigns inside `useEffect`.

2. **Redundant `onClose()` in annotation jump** — `ExplosionOverlay`'s jump button was calling `onClose()` alongside `onAnnotationJump()`. Removed — the overlay closes as a consequence of navigation, not from this button explicitly.

3. **Child chip nested explosion** — Child chips in `ExplosionOverlay` were calling `onAnnotationJump(child.id)` (wrong — scrolls to annotation). Added `onChildClick?: (nodeId: string) => void` prop; `MapCanvas` passes `(nId) => setExplodedNodeId(nId)` to trigger true nested explosion.

4. **ExplosionOverlay accessibility** — Added `role="dialog"`, `aria-modal`, `aria-label`, `tabIndex={-1}` on a hidden sr-only div; `useEffect` focuses it on mount.

**Known debt (not addressed):** `className` coupling between path highlighting (`onNodeMouseEnter`/`onNodeMouseLeave`) and `focus-map-node` event. Both write to ReactFlow node `className` directly via `setNodes`. If a `focus-map-node` fires during hover, the subsequent `mouseLeave` will clear the pulse animation. Requires state-based className computation. Low priority.
